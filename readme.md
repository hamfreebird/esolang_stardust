# **Stardust语言**

---

> Stardust 是一种受 Starry 启发的 esolang 语言，一种类似经典 esolang 中 Whitespace 的语言，程序仅由空格与少量符号构成，指令语义由前导空格数量决定。
与 Starry 不同，Stardust 采用多栈架构：主程序运行在主栈，函数调用时创建独立新栈执行，结束后再将栈内容整体压回主栈，天然支持多值返回。
控制流上，Stardust 保留了基于空格命名的标志与跳转，同时支持通过函数调用来组织逻辑。
在语义上兼顾了结构化与艺术表现力。本解释器使用 Rust 编写。

以下是用 Stardust 编写的 Hello World!

```text
            +               +  *       +* +,
         +            +  *      +** +,            +* +, +,
        +* +,         +             +  * +,        +  *
              + * +,    + +,        +* +,           + * +,
             + *,        +               +  *        +*,"
```

## Stardust 解释器使用说明

### 命令行格式

```bash
stardust <file.stardust|file.sd>                 # 运行 Stardust 程序
stardust --stardust <input.txt> [output]         # 将文本转化为可以打印该文本的 Stardust 源码
stardust --dump <input.stardust> [output]        # 分析一个 Stardust 并输出分析结果
stardust                                         # 不指定任何参数，将启动内置的 IDE，注意内置 IDE 暂不支持带可见 ASCII 字符源码的高亮
```

#### 模式一：运行 Stardust 程序

直接执行 `.stardust` 或 `.sd` 源文件。

```bash
stardust hello.sd
```

解释器将依次进行词法分析、语法解析并启动虚拟机运行。若发生错误，会显示错误类型及源代码位置。

#### 模式二：文本 → Stardust 源码

使用 `--stardust`（或 `-s`）将带可见 ASCII 字符转换为 Stardust 代码，以便于进行编码

```bash
stardust --stardust message.txt                 # 生成 message.stardust
stardust --stardust poem.txt output/poem.sd     # 指定输出路径
```

不保证程序美观，生成的文件可直接用模式一执行，执行后将在终端输出原文本内容。

#### 模式三：Stardust 源码分析

使用 `--dump` ，生成程序的词法分析和语法分析等信息

```bash
stardust --dump code.stardust                               # 生成 code.stardust_dump
stardust --dump code.stardust output/code.stardust_dump     # 指定输出路径
```

#### 文件扩展名要求

- 解释运行模式和分析模式仅接受 `.stardust` （或 `.sd`） 后缀。
- 转换模式输入文件无限制，默认输出文件扩展名为 `.stardust`。

> 为了方便输出文本和数字到终端，代码中的所有可见的 ASCII 字符都会被替换为对应的 Stardust 代码，将该字符的 ASCII 码压入栈中。
> 为了减轻虚拟机计算压力，默认直接压入对应的 ASCII 码，这会在词法分析前进行处理。
> 注意`--stardust` （或 `-s`）的实现和上述不同，它会将代码中的 ASCII 字符转换为美观的，作者倾情手工编写的，错落有致的通过计算得到的 ASCII 码。
> 注释中的 ASCII 字符也会被替换为对应的 Stardust ，这不影响代码的运行。使用 `--stardust` 则会保留注释。

## Stardust 语言的详细介绍

### 一、词法规则

#### 1. 基础元素
程序由以下字符组成：
- 空格 ` `（ASCII 32）
- 符号字符：`+` `*` `` ` `` `'` `:` `;` `.` `,` `~` `/`

其他空白字符（包括换行、制表符）视为 **无效指令** 并被解释器忽略
其他正常字符（除注释中字符）都会视为词法错误

#### 2. 指令格式
每条指令由 **前导空格数量** 加上一个 **符号字符** 构成。
- 前导空格的数量决定 **操作类型** 或 **标识符名称**。
- 符号字符为指令的类别标识。
- 行尾不能出现空格，建议将指令放在完整的一行，注释也是
注释与正常指令不同，由 // 加上注释文本构成。
- 注释只能出现在行尾或占据一整行，范围是从 // 到行尾的 /n 为止

#### 3. 标识符命名
标志名和函数名均使用 **空格数量** 作为唯一标识符。
- 数量 N 可为任意非负整数（N≥0）。
- 同一个数值 N 可同时用作标志名和函数名。

---

### 二、指令集详细定义

以下用 `_` 表示一个空格，用 `(n)` 表示前导空格数量。

#### 1. 栈操作指令（符号 `+`）

| 指令语法    | 空格数   | 操作语义                                      |
|---------|-------|-------------------------------------------|
| `(n) +` | n ≥ 5 | `push (n-5)` —— 将整数 `(n-5)` 压入当前栈         |
| ` +`    | 1     | `dup` —— 弹出栈顶元素 `x`，再压入两次 `x`（栈顶两个 `x`）   |
| `  +`   | 2     | `swap` —— 交换栈顶的两个元素 `a b` → `b a`         |
| `   +`  | 3     | `rotate` —— 将栈顶三个元素循环移位：`a b c` → `c a b` |
| `    +` | 4     | `pop` —— 弹出栈顶元素并丢弃                        |

#### 2. 算术运算指令（符号 `*`）

| 指令语法    | 空格数 | 操作语义                                 |
|---------|-----|--------------------------------------|
| `*`     | 0   | 加法 —— 弹出 `b`，弹出 `a`，压入 `a + b`       |
| ` *`    | 1   | 减法 —— 弹出 `b`，弹出 `a`，压入 `a - b`       |
| `  *`   | 2   | 乘法 —— 弹出 `b`，弹出 `a`，压入 `a * b`       |
| `   *`  | 3   | 除法 —— 弹出 `b`，弹出 `a`，压入 `a / b`（整数除法） |
| `    *` | 4   | 取模 —— 弹出 `b`，弹出 `a`，压入 `a % b`       |

> 注意：减法、除法、取模的顺序为 `a (op) b`，其中 `a` 是先压入栈的值（较深位置），`b` 是栈顶值。

#### 3. 栈整体操作指令（符号 `*`，但空格数不同）

| 指令语法     | 空格数 | 操作语义                                 |
|----------|-----|--------------------------------------|
| `     *` | 5   | `reverse` —— 将当前整个栈反转顺序（栈顶变栈底，栈底变栈顶） |

#### 4. 输入输出指令（符号 `.` 和 `,`）

| 指令语法 | 空格数 | 操作语义                                       |
|------|-----|--------------------------------------------|
| `.`  | 0   | `num_out` —— 弹出栈顶整数，以十进制数字形式输出到标准输出        |
| ` .` | 1   | `num_in` —— 从标准输入读取一个整数，压入当前栈              |
| `,`  | 0   | `char_out` —— 弹出栈顶整数，以其 ASCII 码对应字符输出到标准输出 |
| ` ,` | 1   | `char_in` —— 从标准输入读取一个字符，将其 ASCII 码值压入当前栈  |

> 这里的规定和Starry略有不同，我发现这样在实际的编码中更能体现“有疏有密”的感觉。
> 大多数时候程序接受和输出的都是数字，用.表示整数相关的IO个人感觉更合适;)

#### 5. 控制流指令

| 指令语法      | 空格数  | 操作语义                                               |
|-----------|------|----------------------------------------------------|
| `(n) `` ` | 任意 n | `mark n` —— 在当前代码位置声明一个标志，名称为 `n`                  |
| `(n) '`   | 任意 n | `jump n` —— 当栈中元素不为0时，跳转到标志名称为 `n` 的指令位置继续执行       |
| `(n) ~`   | 任意 n | `UnconditionalJump n` —— 无条件跳转到标志名称为 `n` 的指令位置继续执行 |

> 应当尽量避免使用 ~ 指令，这会让本来就难以阅读和调试的程序变得更加难以阅读和调试

#### 6. 函数声明与调用（符号 `:` 和 `;`）

##### 6.1 函数声明
语法：
```
(n1) : <函数体代码块> :
```
- `(n1)` 为函数名（空格数量）。
- 函数体是一系列指令，直到下一个与声明时相同空格数的冒号 `(n1) :` 为止。

**语义**：  
声明一个函数，名称为 `n1`，函数体为两个冒号之间的指令序列。  
执行时不立即运行，仅记录函数体代码以备调用。

##### 6.2 函数调用
语法：
```
(n1) : (n2) ;
```
- `(n1)` 为要调用的函数名（空格数量）。
- `(n2)` 为传入的参数个数。

**语义**：
1. 从主栈中弹出 `n2` 个值（按从栈顶到栈底的顺序）。
2. 创建一个 **新空栈**（函数执行栈）。
3. 将弹出的 `n2` 个值按 **原来弹出的逆序**（即原栈顶先弹出，但在新栈中应先成为栈底）压入新栈，使得函数内访问参数时顺序与预期一致。
    - 具体：若主栈为 `[... a1 a2 ... aN]`（`aN` 为栈顶），弹出 N 个后得到 `[a1 a2 ... aN]`，再依次压入新栈得到 `[a1 a2 ... aN]`，栈顶为 `aN`。
4. 将当前执行上下文切换到新栈，执行函数体内的指令。
5. 函数体执行完毕后，将新栈中 **所有剩余元素**（可能包括返回值及临时值）依次弹出，并按 **原栈顶先出的顺序** 压回主栈。
    - 即若新栈剩余元素为 `[r1 r2 ... rK]`（`rK` 为栈顶），则主栈被压入 `r1, r2, ..., rK`，使得栈顶为 `rK`。
6. 销毁新栈，恢复主栈执行上下文，继续执行调用指令后的下一条指令。

#### 7. 注释

语法：
```
<代码>  // 注释部分
// 注释部分
```

> 注释只能出现在行尾或占据一整行，范围是从 // 到行尾的 /n 为止。注意代码到注释之间的空白会被计入可能的下一指令的前导空格数中

## Stardust 指令对照表

| 语法（空格数 + 符号）                | Instruction 枚举                 | 操作描述                        | 栈变化（栈顶 → 栈底）                      | 备注          |
|-----------------------------|--------------------------------|-----------------------------|-----------------------------------|-------------|
| **栈操作指令（符号 `+`）**           |
| `(n) +` (n ≥ 5)             | `Push(n-5)`                    | 将整数 `n-5` 压入当前栈             | ` → x`                            | n 为空格数      |
| `_+` (1空格)                  | `Dup`                          | 复制栈顶元素                      | `a → a a`                         | 需栈至少有 1 个元素 |
| `__+` (2空格)                 | `Swap`                         | 交换栈顶两个元素                    | `a b → b a`                       | 需栈至少有 2 个元素 |
| `___+` (3空格)                | `Rotate`                       | 将栈顶三个元素循环右移一位               | `a b c → c a b`                   | 需栈至少有 3 个元素 |
| `____+` (4空格)               | `Pop`                          | 弹出并丢弃栈顶元素                   | `a →`                             | 需栈至少有 1 个元素 |
| **算术运算指令（符号 `*`）**          |
| `*` (0空格)                   | `Add`                          | 加法：弹出 b, 弹出 a，压入 a+b        | `a b → a+b`                       | 需栈至少有 2 个元素 |
| `_*` (1空格)                  | `Sub`                          | 减法：弹出 b, 弹出 a，压入 a-b        | `a b → a-b`                       | 同上          |
| `__*` (2空格)                 | `Mul`                          | 乘法：弹出 b, 弹出 a，压入 a*b        | `a b → a*b`                       | 同上          |
| `___*` (3空格)                | `Div`                          | 整数除法：弹出 b, 弹出 a，压入 a/b      | `a b → a/b`                       | b ≠ 0       |
| `____*` (4空格)               | `Mod`                          | 取模：弹出 b, 弹出 a，压入 a%b        | `a b → a%b`                       | b ≠ 0       |
| `_____*` (5空格)              | `Reverse`                      | 将整个栈反转                      | `[a1...aN] → [aN...a1]`           | 无栈元素数量要求    |
| **输入输出指令（符号 `.` 和 `,`）**    |
| `.` (0空格)                   | `NumOut`                       | 弹出栈顶整数，以十进制输出到 stdout       | `a →`                             | 需栈至少有 1 个元素 |
| `_.` (1空格)                  | `NumIn`                        | 从 stdin 读取整数，压入栈            | ` → x`                            |             |
| `,` (0空格)                   | `CharOut`                      | 弹出栈顶整数，以 ASCII 字符输出         | `a →`                             | a ∈ [0,127] |
| `_,` (1空格)                  | `CharIn`                       | 从 stdin 读取单个字符，压入其 ASCII 码值 | ` → x`                            |             |
| **控制流指令（符号 `` ` `` 和 `'`）** |
| `(n) `` `                   | `Mark { name: n }`             | 声明一个标志，名称为 n（空格数）           | 无栈变化                              | 仅用于解析，运行时跳过 |
| `(n) '`                     | `Jump { name: n }`             | 当栈中元素不为0时，跳转到名称为 n 的标志处     | 无栈变化                              | 标志必须在同一作用域内 |
| `(n) ~`                     | `UnconditionalJump { name: n}` | 无条件跳转到名称为 n 的标志处            | 无栈变化                              | 同上          |
| **函数相关指令（符号 `:` 和 `;`）**    |
| `(n) : ... (n) :`           | 函数声明（不生成运行指令）                  | 定义函数体，名称为 n（空格数）            | 不产生栈操作                            | 解析阶段处理      |
| `(n1) : (n2) ;`             | `Call { name: n1, argc: n2 }`  | 调用函数 n1，传入 n2 个参数           | 从主栈弹出 n2 个参数 → 执行函数 → 将新栈全部内容压回主栈 |             |
| **注释（符号 `//`）**             |
| `// 一段注释`                   | 不生成运行指令                        | 标记一段注释                      | 不产生栈操作                            | 词法分析阶段处理    |

## 指令分类与栈需求速查表

| 分类   | 枚举变体                | 栈需求（最小） | 栈变化（出栈 → 入栈）               |
|------|---------------------|---------|----------------------------|

| 栈顶操作 | `Dup`               | 1       | `a → a a`                  |
|      | `Swap`              | 2       | `a b → b a`                |
|      | `Rotate`            | 3       | `a b c → c a b`            |
|      | `Pop`               | 1       | `a →`                      |
| 算术运算 | `Add`               | 2       | `a b → a+b`                |
|      | `Sub`               | 2       | `a b → a-b`                |
|      | `Mul`               | 2       | `a b → a*b`                |
|      | `Div`               | 2       | `a b → a/b`                |
|      | `Mod`               | 2       | `a b → a%b`                |
| 栈整体  | `Reverse`           | 0       | `[a1..an] → [an..a1]`      |
| I/O  | `NumOut`            | 1       | `a →`                      |
|      | `NumIn`             | 0       | `→ x`                      |
|      | `CharOut`           | 1       | `a →`                      |
|      | `CharIn`            | 0       | `→ x`                      |
| 控制流  | `Mark`              | 0       | 无变化                        |
|      | `Jump`              | 1       | 无变化                        |
|      | `UnconditionalJump` | 0       | 无变化                        |
| 函数   | `Call`              | argc    | 弹出 argc 个参数，执行后可能压入任意数量返回值 |
