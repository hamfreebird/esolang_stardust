# **Stardust语言**

---

> Stardust 是一种受 Starry 启发的 esolang 语言，一种类似经典 esolang 中 Whitespace 的语言，程序仅由空格与少量符号构成，指令语义由前导空格数量决定。
与 Starry 不同，Stardust 采用多栈架构：主程序运行在主栈，函数调用时创建独立新栈执行，结束后再将栈内容整体压回主栈，天然支持多值返回。
控制流上，Stardust 保留了基于空格命名的标志与跳转，同时支持通过函数调用来组织逻辑。
在语义上兼顾了结构化与艺术表现力。本解释器使用 Rust 编写。

以下是用 Stardust 编写的 Hello World!

```text
            +               +  *       +* +,
         +            +  *      +** +,            +* +, +,
        +* +,         +             +  * +,        +  *
              + * +,    + +,        +* +,           + * +,
             + *,        +               +  *        +*,"
```

## Stardust 解释器使用说明

### 命令行格式

```bash
stardust <file.stardust|file.sd>                 # 运行 Stardust 程序
stardust --check <file.sd>                       # 语法检查，输出 JSON 诊断信息
stardust --tokens <file.sd>                      # 输出 Token 流（JSON 格式）
stardust --stardust <input.txt> [output]         # 将文本转化为可以打印该文本的 Stardust 源码
stardust -s <input.txt> [output]                 # 同上（简写形式）
stardust --dump <input.stardust> [output]        # 分析 Stardust 源码并输出分析结果
stardust --help | -h                             # 显示帮助信息
```

#### 模式一：运行 Stardust 程序

直接执行 `.stardust` 或 `.sd` 源文件。

```bash
stardust hello.sd
```

解释器将依次进行词法分析、语法解析并启动虚拟机运行。若发生错误，会显示错误类型及源代码位置。

#### 模式二：文本 → Stardust 源码

使用 `--stardust`（或 `-s`）将带可见 ASCII 字符转换为 Stardust 代码，以便于进行编码

```bash
stardust --stardust message.txt                 # 生成 message.stardust
stardust --stardust poem.txt output/poem.sd     # 指定输出路径
```

不保证程序美观，生成的文件可直接用模式一执行，执行后将在终端输出原文本内容。

#### 模式三：Stardust 源码分析

使用 `--dump` ，生成程序的词法分析和语法分析等信息

```bash
stardust --dump code.stardust                               # 生成 code.stardust_dump
stardust --dump code.stardust output/code.stardust_dump     # 指定输出路径
```

#### 模式四：语法检查（JSON 诊断）

使用 `--check` 对源文件进行语法检查，输出 JSON 格式的诊断信息，适合编辑器/IDE 集成。

```bash
stardust --check code.sd
# 输出示例：
# {"status":"ok","diagnostics":[]}
# 或
# {"status":"error","diagnostics":[{"severity":"error","line":1,"column":1,"message":"...","code":"..."}]}
```

此模式依次进行预处理和词法/语法分析，但不执行程序。错误定位到行列，VSCode 扩展即基于此模式实现实时诊断。

#### 模式五：Token 流输出（JSON）

使用 `--tokens` 输出词法分析的 Token 流，便于调试和工具开发。

```bash
stardust --tokens code.sd
# 输出 JSON：{"tokens": [{"spaces": N, "token_type": "...", "line": L, "column": C, "byte_pos": B}, ...]}
```

#### 文件扩展名要求

- 解释运行模式、`--check`、`--tokens` 和 `--dump` 模式仅接受 `.stardust` （或 `.sd`） 后缀。
- 转换模式输入文件无限制，默认输出文件扩展名为 `.stardust`。

> 为了方便输出文本和数字到终端，代码中的所有可见的 ASCII 字符都会被替换为对应的 Stardust 代码，将该字符的 ASCII 码压入栈中。
> 为了减轻虚拟机计算压力，默认直接压入对应的 ASCII 码，这会在词法分析前进行处理。
> 注意`--stardust` （或 `-s`）的实现和上述不同，它会将代码中的 ASCII 字符转换为美观的，作者倾情手工编写的，错落有致的通过计算得到的 ASCII 码。
> **注意：** 当前美观转换的字符映射尚未完全补全（仅 A-I 有手工映射），其余字母数字字符在 `--stardust` 模式下会输出空串。建议使用默认的 `simple_preprocess` 路径（即模式一运行时的自动替换），其功能完整。
> 注释中的 ASCII 字符也会被替换为对应的 Stardust ，这不影响代码的运行。使用 `--stardust` 则会保留注释。

## Stardust 语言的详细介绍

### 一、词法规则

#### 1. 基础元素
程序由以下字符组成：
- 空格 ` `（ASCII 32）
- 指令符号：`+` `*` `` ` `` `'` `:` `;` `.` `,` `~` `/`
- 保留符号（暂未实现语义）：`-` `=` `<` `>` `&` `#`

其他空白字符（包括换行、制表符）视为 **无效指令** 并被解释器忽略
其他正常字符（除注释中字符）都会视为词法错误

#### 2. 指令格式
每条指令由 **前导空格数量** 加上一个 **符号字符** 构成。
- 前导空格的数量决定 **操作类型** 或 **标识符名称**。
- 符号字符为指令的类别标识。
- 行尾不能出现空格，建议将指令放在完整的一行，注释也是
注释与正常指令不同，由 // 加上注释文本构成。
- 注释只能出现在行尾或占据一整行，范围是从 // 到行尾的 /n 为止

#### 3. 标识符命名
标志名和函数名均使用 **空格数量** 作为唯一标识符。
- 数量 N 可为任意非负整数（N≥0）。
- 同一个数值 N 可同时用作标志名和函数名。

---

### 二、指令集详细定义

以下用 `_` 表示一个空格，用 `(n)` 表示前导空格数量。

#### 1. 栈操作指令（符号 `+`）

| 指令语法    | 空格数   | 操作语义                                      |
|---------|-------|-------------------------------------------|
| `(n) +` | n ≥ 5 | `push (n-5)` —— 将整数 `(n-5)` 压入当前栈         |
| ` +`    | 1     | `dup` —— 弹出栈顶元素 `x`，再压入两次 `x`（栈顶两个 `x`）   |
| `  +`   | 2     | `swap` —— 交换栈顶的两个元素 `a b` → `b a`         |
| `   +`  | 3     | `rotate` —— 将栈顶三个元素循环移位：`a b c` → `c a b` |
| `    +` | 4     | `pop` —— 弹出栈顶元素并丢弃                        |

#### 2. 算术运算指令（符号 `*`）

| 指令语法    | 空格数 | 操作语义                                 |
|---------|-----|--------------------------------------|
| `*`     | 0   | 加法 —— 弹出 `b`，弹出 `a`，压入 `a + b`       |
| ` *`    | 1   | 减法 —— 弹出 `b`，弹出 `a`，压入 `a - b`       |
| `  *`   | 2   | 乘法 —— 弹出 `b`，弹出 `a`，压入 `a * b`       |
| `   *`  | 3   | 除法 —— 弹出 `b`，弹出 `a`，压入 `a / b`（整数除法） |
| `    *` | 4   | 取模 —— 弹出 `b`，弹出 `a`，压入 `a % b`       |

> 注意：减法、除法、取模的顺序为 `a (op) b`，其中 `a` 是先压入栈的值（较深位置），`b` 是栈顶值。

#### 3. 栈整体操作指令（符号 `*`，但空格数不同）

| 指令语法     | 空格数 | 操作语义                                 |
|----------|-----|--------------------------------------|
| `     *` | 5   | `reverse` —— 将当前整个栈反转顺序（栈顶变栈底，栈底变栈顶） |

#### 4. 输入输出指令（符号 `.` 和 `,`）

| 指令语法 | 空格数 | 操作语义                                       |
|------|-----|--------------------------------------------|
| `.`  | 0   | `num_out` —— 弹出栈顶整数，以十进制数字形式输出到标准输出        |
| ` .` | 1   | `num_in` —— 从标准输入读取一个整数，压入当前栈              |
| `,`  | 0   | `char_out` —— 弹出栈顶整数，以其 ASCII 码对应字符输出到标准输出 |
| ` ,` | 1   | `char_in` —— 从标准输入读取一个字符，将其 ASCII 码值压入当前栈  |

> 这里的规定和Starry略有不同，我发现这样在实际的编码中更能体现“有疏有密”的感觉。
> 大多数时候程序接受和输出的都是数字，用.表示整数相关的IO个人感觉更合适;)

#### 5. 控制流指令

| 指令语法      | 空格数  | 操作语义                                               |
|-----------|------|----------------------------------------------------|
| `(n) `` ` | 任意 n | `mark n` —— 在当前代码位置声明一个标志，名称为 `n`                  |
| `(n) '`   | 任意 n | `jump n` —— 当栈中元素不为0时，跳转到标志名称为 `n` 的指令位置继续执行       |
| `(n) ~`   | 任意 n | `UnconditionalJump n` —— 无条件跳转到标志名称为 `n` 的指令位置继续执行 |

> 应当尽量避免使用 ~ 指令，这会让本来就难以阅读和调试的程序变得更加难以阅读和调试

#### 6. 函数声明与调用（符号 `:` 和 `;`）

##### 6.1 函数声明
语法：
```
(n1) : <函数体代码块> :
```
- `(n1)` 为函数名（空格数量）。
- 函数体是一系列指令，直到下一个与声明时相同空格数的冒号 `(n1) :` 为止。

**语义**：  
声明一个函数，名称为 `n1`，函数体为两个冒号之间的指令序列。  
执行时不立即运行，仅记录函数体代码以备调用。

##### 6.2 函数调用
语法：
```
(n1) : (n2) ;
```
- `(n1)` 为要调用的函数名（空格数量）。
- `(n2)` 为传入的参数个数。

**语义**：
1. 从当前栈中弹出 `n2` 个值（按从栈顶到栈底的顺序）。
2. 创建一个 **新调用栈帧**（独立的函数执行栈和指令指针）。
3. 将弹出的 `n2` 个值按 **原来弹出的逆序**（即原栈顶先弹出，但在新栈中应先成为栈底）压入新栈，使得函数内访问参数时顺序与预期一致。
    - 具体：若当前栈为 `[... a1 a2 ... aN]`（`aN` 为栈顶），弹出 N 个后得到 `[a1 a2 ... aN]`，再依次压入新栈得到 `[a1 a2 ... aN]`，栈顶为 `aN`。
4. 将当前执行上下文切换到新栈帧，执行函数体内的指令。
5. 函数体执行完毕后，将新栈帧中 **所有剩余元素**（可能包括返回值及临时值）依次弹出，并按 **原栈顶先出的顺序** 压回调用方栈。
    - 即若新栈剩余元素为 `[r1 r2 ... rK]`（`rK` 为栈顶），则调用方栈被压入 `r1, r2, ..., rK`，使得栈顶为 `rK`。
6. 销毁新栈帧，恢复调用方执行上下文，继续执行调用指令后的下一条指令。

> **支持嵌套与递归调用：** 函数体内可以调用其他函数（包括自身）。解释器使用调用栈帧管理，最大调用深度为 256 层，超过时返回 `CallDepthExceeded` 错误。这天然支持多值返回——函数体内留在栈上的所有值都会传回调用方。

#### 7. 比较运算指令（符号 `=`）

| 指令语法   | 空格数 | 操作语义                                |
|--------|-----|-------------------------------------|
| `=`    | 0   | `eq` —— 弹出 `b` `a`，压入 `(a==b)`（1 或 0） |
| ` =`   | 1   | `ne` —— 弹出 `b` `a`，压入 `(a!=b)`       |
| `  =`  | 2   | `lt` —— 弹出 `b` `a`，压入 `(a<b)`        |
| `   =` | 3   | `gt` —— 弹出 `b` `a`，压入 `(a>b)`        |
| `    =`| 4   | `le` —— 弹出 `b` `a`，压入 `(a<=b)`       |
| `     =`|5   | `ge` —— 弹出 `b` `a`，压入 `(a>=b)`       |

> 比较结果以 1（真）或 0（假）压入栈中，与 `Jump` 联动即可实现 `if-then-else` 条件分支。

#### 8. 逻辑运算指令（符号 `&`）

| 指令语法  | 空格数 | 操作语义                                      |
|-------|-----|-------------------------------------------|
| `&`   | 0   | `and` —— 弹出 `b` `a`，压入 `(a≠0 && b≠0)`（1 或 0） |
| ` &`  | 1   | `or` —— 弹出 `b` `a`，压入 `(a≠0 \|\| b≠0)`      |
| `  &` | 2   | `not` —— 弹出 `a`，压入 `(a==0)`（1 或 0）        |
| `   &`| 3   | `xor` —— 弹出 `b` `a`，压入 `(a≠0 xor b≠0)`    |

> 逻辑运算基于"非零即真"原则，所有结果均为 0 或 1。

#### 9. 堆操作指令（符号 `-`）

| 指令语法 | 空格数 | 操作语义                                     |
|------|-----|------------------------------------------|
| `-`  | 0   | `store` —— 弹出 `addr` `val`，`heap[addr] = val` |
| ` -` | 1   | `load` —— 弹出 `addr`，压入 `heap[addr]`（不存在时为 0） |

> 堆地址和值均为 `i64` 类型。堆空间无限，未写入的地址读取时返回 0。可用于实现变量存储。

#### 10. 栈扩展指令（符号 `<` 和 `>`）

| 指令语法 | 空格数 | 操作语义                                            |
|------|-----|-------------------------------------------------|
| `<`  | 0   | `shiftl` —— 栈循环左移：栈底元素移动到栈顶 `[a1..aN] → [a2..aN, a1]` |
| ` <` | 1   | `depth` —— 压入当前栈的元素数量                             |
| `  <`| 2   | `pick` —— 弹出 `n`，复制 `aN-n`（栈顶下第 n 个）并压入            |
| `>`  | 0   | `shiftr` —— 栈循环右移：栈顶元素移动到栈底 `[a1..aN] → [aN, a1..aN-1]` |
| ` >` | 1   | `dropn` —— 弹出 `n`，丢弃栈顶 n 个元素                      |

> `ShiftL`/`ShiftR` 与 `Reverse`（`     *`）的区别：`Reverse` 完全反转栈，而 `ShiftL`/`ShiftR` 是循环移位。`DropN` 可一次性批量清理栈。

#### 11. 调试指令（符号 `#`）

| 指令语法 | 空格数 | 操作语义                                           |
|------|-----|------------------------------------------------|
| `#`  | 0   | `dumpstack` —— 将当前栈全部内容输出到 stderr（格式：`[a1, a2, ...]`） |
| ` #` | 1   | `dumpstate` —— 将 PC、栈深度、堆大小等解释器状态输出到 stderr       |
| `  #`| 2   | `breakpoint` —— 调试断点（当前为 NOP，预留 `--debug` 模式使用）    |

> 调试指令输出到 **stderr**，不影响程序正常输出流（stdout）。

#### 12. 注释

语法：
```
<代码>  // 注释部分
// 注释部分
```

> 注释只能出现在行尾或占据一整行，范围是从 // 到行尾的 /n 为止。注意代码到注释之间的空白会被计入可能的下一指令的前导空格数中

| **输入输出指令（符号 `.` 和 `,`）**    |
| `.` (0空格)                   | `NumOut`                       | 弹出栈顶整数，以十进制输出到 stdout       | `a →`                             | 需栈至少有 1 个元素 |
| `_.` (1空格)                  | `NumIn`                        | 从 stdin 读取整数，压入栈            | ` → x`                            |             |
| `,` (0空格)                   | `CharOut`                      | 弹出栈顶整数，以 ASCII 字符输出         | `a →`                             | a ∈ [0,127] |
| `_,` (1空格)                  | `CharIn`                       | 从 stdin 读取单个字符，压入其 ASCII 码值 | ` → x`                            |             |
| **控制流指令（符号 `` ` `` 和 `'`）** |
| `(n) `` `                   | `Mark { name: n }`             | 声明一个标志，名称为 n（空格数）           | 无栈变化                              | 解析阶段验证引用，运行时跳过 |
| `(n) '`                     | `Jump { name: n }`             | 弹出栈顶，非零时跳转到名称为 n 的标志处      | 弹出 1 个元素                          | 标志引用在解析阶段验证（不再仅运行时检查） |
| `(n) ~`                     | `UnconditionalJump { name: n}` | 无条件跳转到名称为 n 的标志处            | 无栈变化                              | 标志引用在解析阶段验证          |
| **函数相关指令（符号 `:` 和 `;`）**    |
| `(n) : ... (n) :`           | 函数声明（不生成运行指令）                  | 定义函数体，名称为 n（空格数）            | 不产生栈操作                            | 解析阶段处理      |
| `(n1) : (n2) ;`             | `Call { name: n1, argc: n2 }`  | 调用函数 n1，传入 n2 个参数           | 从当前栈弹出 n2 个参数 → 压入新调用帧 → 执行函数 → 将新帧栈内容压回调用方 | 支持嵌套/递归，最大深度 256 |
| **比较运算指令（符号 `=`）**          |
| `=` (0空格)                   | `Eq`                            | 相等比较：弹出 b, a，压入 (a==b)       | `a b → 0\|1`                        | 需栈至少 2 元素 |
| ` =` (1空格)                  | `Ne`                            | 不等比较：弹出 b, a，压入 (a!=b)       | `a b → 0\|1`                        | 同上 |
| `  =` (2空格)                 | `Lt`                            | 小于比较：弹出 b, a，压入 (a<b)        | `a b → 0\|1`                        | 同上 |
| `   =` (3空格)                | `Gt`                            | 大于比较：弹出 b, a，压入 (a>b)        | `a b → 0\|1`                        | 同上 |
| `    =` (4空格)               | `Le`                            | 小于等于：弹出 b, a，压入 (a<=b)       | `a b → 0\|1`                        | 同上 |
| `     =` (5空格)              | `Ge`                            | 大于等于：弹出 b, a，压入 (a>=b)       | `a b → 0\|1`                        | 同上 |
| **逻辑运算指令（符号 `&`）**          |
| `&` (0空格)                   | `And`                           | 逻辑与：弹出 b, a，压入 (a&&b)        | `a b → 0\|1`                        | 需栈至少 2 元素 |
| ` &` (1空格)                  | `Or`                            | 逻辑或：弹出 b, a，压入 (a\|\|b)      | `a b → 0\|1`                        | 同上 |
| `  &` (2空格)                 | `Not`                           | 逻辑非：弹出 a，压入 (!a)            | `a → 0\|1`                          | 需栈至少 1 元素 |
| `   &` (3空格)                | `Xor`                           | 逻辑异或：弹出 b, a，压入 (a xor b)   | `a b → 0\|1`                        | 需栈至少 2 元素 |
| **堆操作指令（符号 `-`）**           |
| `-` (0空格)                   | `Store`                         | 堆存储：弹出 addr, val, heap[addr]=val | `addr val →`                         | 需栈至少 2 元素 |
| ` -` (1空格)                  | `Load`                          | 堆加载：弹出 addr，压入 heap[addr]     | `addr → heap[addr]`                  | 需栈至少 1 元素 |
| **栈扩展指令（符号 `<` 和 `>`）**    |
| `<` (0空格)                   | `ShiftL`                        | 循环左移：[a1..aN]→[a2..aN,a1]     | `[a1..aN] → [a2..aN, a1]`           | 需栈至少 1 元素 |
| ` <` (1空格)                  | `Depth`                         | 压入栈元素数量                       | `→ n`                                | 无栈要求 |
| `  <` (2空格)                 | `Pick`                          | 弹出 n，复制 aN-n 压入              | `n → aN-n`                           | 需栈至少 n+1 元素 |
| `>` (0空格)                   | `ShiftR`                        | 循环右移：[a1..aN]→[aN,a1..aN-1]   | `[a1..aN] → [aN, a1..aN-1]`          | 需栈至少 1 元素 |
| ` >` (1空格)                  | `DropN`                         | 弹出 n，丢弃栈顶 n 个元素              | `n →`                                | 需栈至少 n 元素 |
| **调试指令（符号 `#`）**            |
| `#` (0空格)                   | `DumpStack`                     | 将栈内容输出到 stderr                | 无栈变化                               | 输出 `[a1, a2, ...]` |
| ` #` (1空格)                  | `DumpState`                     | 将解释器状态输出到 stderr              | 无栈变化                               | 输出 PC/栈深度/堆大小 |
| `  #` (2空格)                 | `Breakpoint`                    | 调试断点（当前 NOP）                   | 无栈变化                               | 预留 --debug 模式 |
| **注释（符号 `//`）**             |
| `// 一段注释`                   | 不生成运行指令                        | 标记一段注释                      | 不产生栈操作                            | 词法分析阶段处理    |

## 指令分类与栈需求速查表

| 分类   | 枚举变体                | 栈需求（最小） | 栈变化（出栈 → 入栈）               |
|------|---------------------|---------|----------------------------|
| 数据压入 | `Push(v)`           | 0       | `→ v`                      |
| 栈顶操作 | `Dup`               | 1       | `a → a a`                  |
|      | `Swap`              | 2       | `a b → b a`                |
|      | `Rotate`            | 3       | `a b c → c a b`            |
|      | `Pop`               | 1       | `a →`                      |
| 算术运算 | `Add`               | 2       | `a b → a+b`                |
|      | `Sub`               | 2       | `a b → a-b`                |
|      | `Mul`               | 2       | `a b → a*b`                |
|      | `Div`               | 2       | `a b → a/b`                |
|      | `Mod`               | 2       | `a b → a%b`                |
| 栈整体  | `Reverse`           | 0       | `[a1..an] → [an..a1]`      |
| I/O  | `NumOut`            | 1       | `a →`                      |
|      | `NumIn`             | 0       | `→ x`                      |
|      | `CharOut`           | 1       | `a →`                      |
|      | `CharIn`            | 0       | `→ x`                      |
| 控制流  | `Mark`              | 0       | 无变化                        |
|      | `Jump`              | 1       | 无变化                        |
|      | `UnconditionalJump` | 0       | 无变化                        |
| 函数   | `Call`              | argc    | 弹出 argc 个参数，执行后可能压入任意数量返回值 |
| 比较   | `Eq` `Ne` `Lt` `Gt` `Le` `Ge` | 2       | `a b → 0\|1`                 |
| 逻辑   | `And` `Or` `Not` `Xor`         | 2 (Not: 1) | `a b → 0\|1`               |
| 堆     | `Store` `Load`                  | 2 / 1   | `addr val →` / `addr → val`  |
| 栈扩展  | `ShiftL` `Depth` `Pick` `ShiftR` `DropN` | 0-1 | 见指令表               |
| 调试   | `DumpStack` `DumpState` `Breakpoint` | 0       | 无栈变化（输出到 stderr）         |
